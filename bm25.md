## BM25 Explanation for the Implementation in bsbi.py

This file explains the BM25 scoring logic implemented in [bsbi.py](bsbi.py#L226), specifically inside the retrieve_bm25 function.

## Where It Is Implemented

- Main function: [retrieve_bm25](bsbi.py#L226)
- Default parameters: k = 10, k1 = 1.625, b = 0.75

## What is k, k1, and b?
### 1) k (Top-k size)
- Meaning: number of final documents returned.
- Practical effect:
	- Smaller k: usually faster queries and stricter output size.
	- Larger k: more results, but usually more work and slightly slower.


### 2) k1 (BM25 TF saturation)
- Meaning: controls how strongly term frequency tf increases score.
- Practical effect:
	- Lower k1: tf saturates faster (extra repetitions of a term add less gain).
	- Higher k1: tf has stronger influence before saturating.

Relevant code:
~~~python
numerator = tf * (k1 + 1)
denominator = tf + k1 * (1 - b + b * (doc_len / avgdl))
~~~

### 3) b (Length normalization strength)

- Meaning: controls document-length normalization in BM25.
- Range in BM25 practice: typically between 0 and 1.
- In this implementation:
	- b = 0 means almost no length normalization.
	- b = 1 means full normalization by doc_len / avgdl.
- Practical effect:
	- Higher b penalizes long documents more strongly.
	- Lower b reduces that penalty.

Relevant code:

~~~python
denominator = tf + k1 * (1 - b + b * (doc_len / avgdl))
~~~


## Core Formula Used

For each query term t and document D, the score contribution is:

$$
score(D, Q) = \sum_{t \in Q} IDF(t) \cdot \frac{tf(t, D) \cdot (k_1 + 1)}{tf(t, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{avgdl}\right)}
$$

The IDF formula used in this implementation is:

$$
IDF(t) = \ln\left(\frac{N - df(t) + 0.5}{df(t) + 0.5} + 1\right)
$$

Variable meanings:

- N: total number of documents in the collection
- df(t): number of documents that contain term t
- tf(t, D): term frequency of t in document D
- |D|: length of document D
- avgdl: average document length across the collection
- k1: term-frequency saturation parameter
- b: document-length normalization parameter

## Code Excerpts from the Actual Implementation

1) Function and default BM25 parameters

~~~python
def retrieve_bm25(self, query, k=10, k1=1.625, b=0.75):
~~~

2) Query term filtering (ignore out-of-vocabulary terms)

~~~python
terms = []
for word in query.split():
    if word in self.term_id_map.str_to_id:
        terms.append(self.term_id_map[word])
~~~

3) Collection statistics used by BM25

~~~python
N = len(merged_index.doc_length)
total_length = sum(merged_index.doc_length.values())
avgdl = total_length / N if N > 0 else 1
~~~

4) IDF computation used in the scoring loop

~~~python
df = merged_index.postings_dict[term][1]
idf = math.log((N - df + 0.5) / (df + 0.5) + 1)
~~~

5) BM25 term contribution per document

~~~python
numerator = tf * (k1 + 1)
denominator = tf + k1 * (1 - b + b * (doc_len / avgdl))
scores[doc_id] += idf * (numerator / denominator)
~~~

6) Final ranking and top-k return

~~~python
docs = [(score, self.doc_id_map[doc_id]) for (doc_id, score) in scores.items()]
return sorted(docs, key=lambda x: x[0], reverse=True)[:k]
~~~

## Step-by-Step Execution Flow

1. The function ensures dictionaries are loaded.
If term_id_map or doc_id_map is empty, it calls self.load().

2. It tokenizes the query by whitespace.
Only known terms are kept for scoring.

3. It opens the merged inverted index.
Then it computes N and avgdl from document statistics.

4. For each query term in the index:
- Get document frequency df and compute IDF.
- Retrieve postings and tf_list.
- For each document in postings, compute BM25 contribution and accumulate score.

5. It converts doc_id to document path and sorts by score descending.
Only the top k documents are returned.

## Implementation Notes

- Unknown query terms are skipped safely.
- Scores are accumulated in a dictionary keyed by doc_id.
- Output format is a list of pairs: score and document path.
- The implementation is Term-at-a-Time style scoring.

## Intuition

- Higher tf increases score, but with diminishing returns.
- Longer documents are normalized through |D|/avgdl.
- Rare terms get larger IDF weight and influence ranking more strongly.
